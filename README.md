# Pharmacy E2E Tests

Playwright E2E test suite for NHS pharmacy websites. Covers the full patient journey:

`/conditions` → condition detail → eligibility check → questionnaire → NHS PDS sign-up / Private booking → payment → thank you

Supports multiple pharmacy sites as separate Playwright **projects**, and includes a local **dashboard** for running tests, editing test data, and viewing results from a browser UI.

---

## Requirements

- [Node.js](https://nodejs.org/) 18+
- npm / pnpm
- Chromium (installed via Playwright)

---

## Setup

```bash
npm install
npx playwright install chromium
```

No `.env` changes are required for the predefined pharmacy sites. Optionally copy `.env.example` to `.env` if you need to override with a custom URL:

```bash
cp .env.example .env
# Uncomment and edit BASE_URL= if needed
```

## Running Tests

### npm scripts

| Command | Description |
|---|---|
| `npm test` | Run all tests headless |
| `npm run test:headed` | Run with a visible browser window |
| `npm run test:debug` | Step through each action interactively |
| `npm run test:ui` | Open Playwright UI mode |
| `npm run test:report` | View the last HTML report |

### Run against a specific pharmacy

```bash
npm run test:pharmaease     # Pharmaease
npm run test:paydens        # Paydens Pharmacy
npm run test:pharmacist     # The Pharmacist
npm run test:localhost      # http://localhost:4005
```

Or use `--project` directly:

```bash
npx playwright test --project="Pharmaease"
```

### One-off CI / custom URL

```bash
BASE_URL=https://example.healthya.co.uk npx playwright test
```

When `BASE_URL` is set, all pharmacy projects are replaced by a single `CI Override` project targeting that URL.

## Dashboard

A local web dashboard lets you configure and run tests without touching the command line.

```bash
npm run dashboard
# Opens at http://localhost:7890
```

Features:
- **Select pharmacy** and **test file / individual test** to run
- **Live log stream** via SSE while tests are running
- **Edit test data** (user, payment, booking preferences) from the browser — changes are written back to `tests/fixtures/test-data.ts`
- **View videos and traces** generated by the last run
- **Launch Playwright UI** at `http://localhost:8081`

### Codespaces / headless servers

Use the helper script to get a live browser preview inside GitHub Codespaces (or any headless Linux box):

```bash
bash start-codespaces.sh
```

This starts Xvfb + VNC + noVNC, then launches the dashboard. Forward ports **6080** (live browser) and **7890** (dashboard) in Codespaces.

## Test Flows

### Condition journey types

Set `ACTIVE_CONDITION.journeyType` in `tests/fixtures/test-data.ts` (or via the dashboard):

| Type | Condition | Notes |
|---|---|---|
| `nhs` | Shingles | NHS PDS lookup → appointment booking |
| `private` | Weight Management | Card payment → appointment booking |
| `lifestyle` | Erectile Dysfunction | Drug selection → cart → checkout |

### Predefined flow configs (`tests/fixtures/flow-configs.ts`)

| Flow | Type | Booking |
|---|---|---|
| NHS — next available slot | NHS | Auto-select first slot |
| NHS — specific date and time | NHS | Pick month / date / time |
| Private — next available slot, new card | Private | Auto-select, pay with new card |
| Private — next available slot, saved card | Private | Auto-select, pay with saved card |
| Private — specific date, new card | Private | Pick slot, pay with new card |
| Private — specific date, saved card | Private | Pick slot, pay with saved card |

## Troubleshooting

**Tests time out on `/conditions`**
Ensure the target app is running and reachable. Check `BASE_URL` or the pharmacy `baseURL` in `pharmacies.ts`.

**NHS PDS check fails / no match**
Expected — the test user (John Smith, SW1A 1AA, 01/01/1990) is unlikely to match a real NHS record. The flow continues via the manual sign-up path.

**Videos and traces not found**
Artifacts are saved to `test-results/`. Run `npm run test:report` to open the HTML report, or browse artifacts from the dashboard after a run.
